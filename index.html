git branch -v

Komenda listuje wszystkie dostępne gałęzie w danym repozytorium. Jest bardzo przydatna w przypadku, gdy osoba posiada wiele gałęzi nad którymi pracuje i chce odnaleźć tę, która go aktualnie interesuje.

  • v - wyświetla dodatkowe informacje na temat ostatnich commitów wykonanych na poszczególnych gałęziach,

git reset HEAD

  • cofa zmiany, wracając do plików wyjściowych,
  • można regulować ile zrobionych przez nas commitów ma być cofniętych np git reset HEAD^ , cofnie nam o jeden, natomiast git reset HEAD~2 o dwa,
  • gdy po modyfikowaniu, usuwaniu i dodawaniu wierszy w plikach, okazuje się, że zmiany nie są dobre i chcemy wrócić do wyjściowego pliku git reset HEAD index.html ,

git remote add "remote_name" "git url"

Komenda służąca do dodania nowego zdalnego repozytorium, która przyjmuje dwa argumenty: "remote_name", czyli nazwę zdalnego repozytorium oraz url do niego "git url".
Przykład: git remote add "nazwa" "https://github.com/user/repo.git"

git commit -a -m ""

Zatwierdza zmodyfikowane i usunięte pliki w danym repozytorium.

  • a - (all - wszystkie) - parametr, dzięki któremu zatwierdzone zostaną wszystkie zmiany,
  • m - (message - wiadomość) - parametr, którty oznacza dodanie do "commita" krótkiej wiadomości,
  • "" - miejsce na krótką wiadomość,

git checkout "branch"

Komenda git checkout pozwala zmienić gałąź na której pracujemy.
Wywołanie komendy git checkout "branch" przełączy nas na gałąź o nazwie "branch" i zaktualizuje pliki w katalogu roboczym na te z wybranej gałęzi.
Jeśli gałąź o nazwie "branch" nie istnieje, komenda nie zadziała. Aby stworzyć nową gałąź o tej nazwie należy poprzedzić nazwę gałęzi flagą -b (git checkout -b "branch") .

git blame -C "filename"

Komenda poprzedza każdą linię w pliku "filename" informacją o miejscu, autorze oraz czasie ostatniej modyfikacji danej linii. Nie podaje jednak informacji na temat usuniętych oraz zastąpionych linii.
Dzięki opcji -C wykrywa linie skopiowane z innych plików w repozytorium.

Przykładowy wynik komendy działania git blame -C index.html :
a0ddc9f8 (Patryk Żywica 2020-11-10 19:37:44 +0100 1) <html>
a0ddc9f8 (Patryk Żywica 2020-11-10 19:37:44 +0100 2) <body> pusta strona </body>
a0ddc9f8 (Patryk Żywica 2020-11-10 19:37:44 +0100 3) </html>

git branch "name"

Komenda git branch za argument przyjmuje ciąg znaków i tworzy nowy branch o podanej nazwie "name". Może być przydatna np. gdy podczas prac nad działającą aplikacją nie chcemy jej zepsuć, co umożliwia nam stworzenie nowego brancha dedykowanego wprowadzanym zmianom.

git config --global user.name ""

Komenda służy do wykonania pierwszych konfiguracji gita. Wystarczy dokonać tej konfiguracji tylko raz (informacje zostaną zachowane nawet po aktualizacji). Za pomocą parametrów, które są wykorzystywane powyżej następuje przypisanie wszystkich tworzonych zmian do konkretnego użytkownika.

  • Paramter --global jest opcjonalny, jednak to on odpowiada za to, że dane pozostają nienaruszone po zmianach w aplikacji (np. po aktualizacji).
  • Parametr user.name mówi o tym, co zamierzamy skonfigurować - w tym przypadku jest to nazwa użytkownika.
  • "" w cudysłowie należy podać wartość, która jest nazwą użytkownika, z którą chcemy być identyfikowani w dalszej części pracy nad projektem.

git config --global color.ui true

Jest głównym przełącznikiem pozwalającym konfigurować koloryzowanie danych wyjścia.
Opcja:

  • auto - ustawiona jest domyślnie, koloruje jedynie dane wyjściowe trafiające prosto do terminalu,
  • true - włącza pełne koloryzowanie danych wyjściowych trafiających zarówno do terminala jak i np. pliku,
  • false - wyłącza opcję koloryzowania danych wyjściowych,

git diff --staged

Komenda pokazuje różnice między katalogiem, w którym aktualnie się znajdujemy a listą plików, które mają zostać scommitowane.

  • opcja --staged działa tak samo jak opcja --cached - dzięki niej komenda git diff pokazuje różnice między plikami, które mają zostać scommitowane a tymi, które znajdują się w Head,

git log --graph --all

Komenda git log wywołana bez argumentów listuje zmiany zatwierdzone w repozytorium w odwrotnej kolejności chronologicznej.
Opcje:

  • --graph - wyświetla graf, obrazujący gałęzie repozytorium i ich połączenia, wraz z informacjami o commitach,
  • --all - wyświetla historię wszystkich zmian w repozytorium,
